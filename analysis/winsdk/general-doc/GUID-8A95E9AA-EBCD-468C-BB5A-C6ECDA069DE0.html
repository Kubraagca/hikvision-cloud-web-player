Encrypt HTTPS Link

When the streaming mode is RTP over HTTPS, that is, the stream with RTP container format is transmitted on the link based on HTTPS, you can encrypt the HTTPS link to ensure the security of stream transmission.

  1. Call NET_DVR_GetDeviceAbility and set the capability type to "DEVICE_SOFTHARDWARE_ABILITY" (macro definition value: 0x001) for getting the device software and hardware capability (XML_BasicCapability) to check if encrypting HTTPS link is supported by device.

    If supported, the node <RtspoverHttpsSupport> will be returned in the capability and its value is "1".

  2. Enable HTTPS link encryption for live view or playback.

    • Call NET_DVR_RealPlay_V40 and set dwLinkMode and byProtoType of live view parameter structure (NET_DVR_PREVIEWINFO) to "7" and "1", respectively, for enabling HTTPS link encryption during live view.

    • Call NET_DVR_PlayBackByTime_V50 or NET_DVR_PlayBackByName to automatically enable HTTPS link encryption without setting any parameters during playback.

  3. Call NET_DVR_RealPlay_V40 or NET_DVR_PlayBackByTime_V50/NET_DVR_PlayBackByName to automatically enable HTTPS link encryption without setting any parameters during live view or playback.
